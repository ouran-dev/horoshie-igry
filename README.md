# Хорошие игры

Детские мини-игры от **Хорошево** для Windows (сенсорные панели, x64).  
Все зависимости **вшиты** в установщик — отдельно ставить .NET на панель не нужно.

---

## Что нужно на компьютере разработчика (один раз)

1. **Windows 10/11 x64**
2. **[.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)** — для сборки
3. **Git** — для работы с репозиторием
4. **Инструмент Velopack `vpk`** — ставится автоматически при первой сборке релиза:
   ```bat
   dotnet tool install -g vpk --version 1.2.0
   ```
5. **Токен GitHub** (для публикации обновлений) — см. ниже

Для обычной разработки без релиза достаточно SDK:

```bat
Запуск.bat
```

или

```bat
dotnet run
```

| Параметр | Значение |

| GitHub | https://github.com/ouran-dev/horoshie-igry |

| **Последний релиз** | https://github.com/ouran-dev/horoshie-igry/releases/latest |

| Настройки в коде | `Core/Updates/UpdateSettings.cs` |

Установленная **онлайн-версия** (`Setup.exe`) сама проверяет GitHub Releases.  
В игре: **Настройки → «Проверить обновления»**. При запуске новая версия тоже может предложить перезапуск.

## Сборка релиза (на вашем ПК)

### 1. Увеличьте версию

В `HoroshieIgry.csproj`:

```xml
<Version>1.0.5</Version>
<AssemblyVersion>1.0.5.0</AssemblyVersion>
<FileVersion>1.0.5.0</FileVersion>
```

### 2. Запустите сборку

```bat
Сборка-Релиза.bat
```

Или с явной версией:

```bat
Сборка-Релиза.bat 1.0.5
```

Скрипт:
- создаёт иконку `Assets\Brand\AppIcon.ico` из `logo.png`;
- собирает **онлайн** и **офлайн** версии (self-contained, win-x64);
- упаковывает `Releases\Setup.exe` (Velopack, **с автообновлением**);
- собирает zip **офлайн-версии** для флешки;
- при наличии токена — публикует на GitHub.

### 3. Публикация на GitHub

Создайте [Personal Access Token](https://github.com/settings/tokens) с правом `repo` (или fine-grained: Contents Read/Write для репозитория).

В **cmd** (не коммитьте токен в git):

```bat
set GITHUB_TOKEN=ghp_ваш_токен
Сборка-Релиза.bat
```

Либо через Git tag (сработает GitHub Actions):

```bat
git tag v1.0.5
git push origin v1.0.5
```

## Установка на новую панель

### Вариант A — с автообновлением (основной)

1. Скопируйте с флешки или скачайте с GitHub Releases файл **`Setup.exe`**.
2. Запустите установщик.
3. Выберите **папку установки** (Velopack спросит при установке).
4. При **первом запуске** игра спросит, **создать ли ярлык** и **в какую папку** его положить.
5. Дальнейшие обновления — через **Настройки** или автоматически при старте.

### Вариант B — офлайн, без автообновления (запасной)

Подходит для флешки, если интернет на панели нестабилен.

1. Распакуйте **`HoroshieIgry-Offline-X.X.X-win-x64.zip`**  
   (из `Releases\` после сборки или с GitHub Releases).
2. Запустите **`Установить-офлайн.bat`**.
3. В окне установки:
   - выберите **папку установки**;
   - отметьте **«Создать ярлык»**;
   - укажите **папку для ярлыка** (по умолчанию — рабочий стол).
4. В настройках будет написано, что это офлайн-версия без автообновления.

Чтобы обновить офлайн-панель — установите новый zip поверх или заново.

## Структура после сборки

```
Releases/
  Setup.exe                              ← онлайн, автообновление
  HoroshieIgry-Offline-1.0.4-win-x64.zip ← офлайн, для флешки
  Offline/
    HoroshieIgry-1.0.4-win-x64/          ← распакованный офлайн-пакет
      HoroshieIgry.exe
      Установить-офлайн.bat
      ...
```

Папки `publish/`, `publish-offline/`, `Releases/` в git **не коммитятся** (см. `.gitignore`).

## Что положить на флешку для новой панели

| Файл | Назначение |
|------|------------|
| `Setup.exe` | Обычная установка + обновления с GitHub |
| `HoroshieIgry-Offline-….zip` | Запасной вариант без интернета |

Исходный код на флешку для установки **не нужен** — только установщики из `Releases\`.

## Частые вопросы

**«Проверить обновления» неактивно**  
Вы запустили игру через `dotnet run` или офлайн-сборку. Нужна установка через `Setup.exe`.

**Нет иконки при установке**  
Положите `logo.png` в `Assets\Brand\` (или в корень проекта) и пересоберите — скрипт создаст `AppIcon.ico`.

**Ошибка при `vpk upload`**  
Проверьте `GITHUB_TOKEN`, доступ к репозиторию `ouran-dev/horoshie-igry` и что версия в `.csproj` новее предыдущего релиза.
